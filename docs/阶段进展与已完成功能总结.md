## 2. 数据库与后端基础设施

- 已完成：
  - 本地默认使用 SQLite（`dev.db`），并保留 MySQL（`tourism_imu`）的生产接入能力（通过 `DATABASE_URL` 切换）：
    - `user`：用户账号信息（用户名、密码哈希、邮箱、性别、地区等）；
    - `user_profile`：用户画像（偏好景点类型、美食类型、出行风格、预算等级、出行频率等）；
    - `user_behavior_log`：用户行为日志（浏览、点击、评分行为等）；
    - `rating`：通用评分表（对景点/酒店/美食的打分与评论）；
    - `scenic_spot`：景点资源；
    - `hotel`：酒店/住宿资源；
    - `food_place`：美食/餐饮资源；
    - `content_standard`：标准化内容与指标（受欢迎度、情感/口碑得分、协同过滤相似度等）；
    - `trip` / `trip_day` / `trip_item`：登录用户保存的单日与多日行程、每日节点与项目快照。
  - 通过 SQLAlchemy 建立了与上述表一一对应的 ORM 模型，并在 `models.py` 中集中管理。
  - 应用启动时会通过 `db.create_all()` 非破坏性补齐缺失表，当前已用于 trip 系列表的落地。
  - 创建了虚拟环境 `venv`，并完成 Flask、SQLAlchemy、PyMySQL 等依赖安装与配置。

- 现有文档与脚本：
  - 《数据库建表.sql》：完整的建表脚本；
  - `backend/app/models.py`：所有 ORM 定义；
  - `backend/app/config.py`：数据库连接、地图 API key 等配置项。

## 3. 后端业务接口与服务模块

### 3.1 基础服务

- **健康检查**：`GET /api/health`，用于快速确认后端服务是否正常。
- **用户认证**：
  - `POST /api/auth/register`：注册接口，包含用户名/密码校验与唯一性检查；
  - `POST /api/auth/login`：登录接口，返回用户基础信息。

### 3.2 资源浏览接口

- **景点列表**：`GET /api/scenic-spots`，支持分页、城市筛选、类型筛选、关键字搜索，按评分+人气排序。
- **酒店列表**：`GET /api/hotels`，支持城市、星级、价格区间和关键字筛选。
- **美食列表**：`GET /api/foods`，支持城市、菜系、人均价区间和关键字筛选。

### 3.3 行为与评分接口

- **行为日志**：`POST /api/behaviors`，记录用户浏览、点击等行为，为画像与推荐提供数据基础。
- **评分评价**：`POST /api/ratings`，对景点/酒店/美食打分并评论，同时更新目标实体的平均分和评分次数。

### 3.4 推荐与画像相关接口

- **景点推荐（多源 + 画像 + CF/MF/Hybrid）**：`GET /api/recommend/scenic-spots`
  - 未登录或无画像：按热门 + 多源融合排序；
  - 有画像：根据 `user_profile.prefer_scenic_types` 调整排序；
  - 支持 `cf/mf/hybrid` 个性化策略与回退。
- **酒店推荐**：`GET /api/recommend/hotels`（热门排序）；
- **美食推荐**：`GET /api/recommend/foods`（热门排序）。
- **用户画像读取/保存**：
  - `GET /api/user/profile/{user_id}`：获取指定用户画像与手动偏好；
  - `POST /api/user/profile`：创建或更新用户的出行风格、预算等级、显式偏好类型等。
- **相似景点推荐（协同过滤）**：`GET /api/recommend/scenic-spots/{spot_id}/similar`
  - 基于离线脚本 `aggregate_scenic_cf.py` 生成的 item-based CF 相似度，返回与某景点相似的一组景点。

### 3.6 自然语言 / 个性化地图 / 路线方案（新增）

- **NLU 解析**：`POST /api/nlu/parse`（意图 + 槽位）
- **个性化地图**：`POST /api/map/personalized`
- **偏好对话 Agent**：`POST /api/agent/chat`（抽取 slots、可写入画像并生成行程草案）
- **路线方案**：`POST /api/route/options` + 指标 `GET /api/route/metrics`

### 3.5 行程与路线规划接口

- **路线规划**：`POST /api/route/plan`
  - 输入：若干景点 ID 列表，可选起点 ID；
  - 输出：按游览顺序排列的景点列表 + 估算总里程（基于经纬度的最近邻启发式）；
  - 如配置 `AMAP_WEB_KEY`，可调用高德驾车路径规划 API 优化总里程。
- **我的行程 CRUD**：
  - `GET /api/trips?user_id=...`：查看当前用户的行程摘要列表；
  - `GET /api/trips/<trip_id>?user_id=...`：查看单个行程详情；
  - `POST /api/trips`：统一创建 trip，兼容 `/route` 单日路线保存、`/profile` Agent 多日草案落库与 `/trips/new` 空白草稿创建；
  - `PUT /api/trips/<trip_id>`：整单更新标题、日期、风格、多日备注与 item 顺序/时间/快照字段；
  - `DELETE /api/trips/<trip_id>?user_id=...`：删除行程并级联删除子表数据。

- 所有上述接口均已在《接口与模块设计.md》中详细记录。

## 4. 离线聚合与推荐算法实现

- **内容受欢迎度聚合（aggregate_content.py）**
  - 基于 `rating` 和资源表，计算内容的综合热门度，写入 `content_standard.popularity_score`.

- **用户画像聚合（aggregate_user_profile.py）**
  - 根据 `rating` 中用户对景点/美食的评分，统计偏好类型与评分次数；
  - 更新 `user_profile` 中的 `prefer_scenic_types`、`prefer_food_types`、`travel_frequency` 等字段.

- **协同过滤相似度聚合（aggregate_scenic_cf.py）**
  - 基于景点评分数据构建 item-based 协同过滤相似度；
  - 将每个景点的相似邻居列表写入 `content_standard`（`source_type = "item_cf_similarity"`）；
  - 为 `/api/recommend/scenic-spots/{spot_id}/similar` 提供数据支撑.

- **多源数据采集与融合**（已完成 OTA 首版，尚未接入真实大规模社交媒体数据）：
  - OTA：`backend/data/ota_sample.csv` + `OTACollector`（`app/pipelines/collectors/ota_collector.py`）+ `load_ota_data.py`，从真实 OTA 页面整理的 70 条数据写入 `content_standard`（`source_type = "ota_stats"`，兼容历史 `ota`；读取时兼容 `external_rating/rating` 与 `review_count/external_review_count`）；
  - 社交媒体：`social_media_fetcher.py` 作为社交媒体与 UGC 数据采集骨架，后续可在此基础上接入微博/小红书等数据源.

## 5. 前端应用与用户体验

- 前端技术栈：Vue 3 + Vite + Vue Router + Axios.
- 目录：`E:\旅游\frontend`。

### 5.1 已实现的主要页面

- **首页 / 推荐页**（`/`，`HomeView.vue`）：
  - 展示景点、酒店、美食推荐；
  - 已登录用户可看到基于画像调整后的景点推荐；
  - 提供“刷新”按钮触发重新拉取推荐结果。

- **目的地浏览页**（`/browse`，`BrowseView.vue`）：
  - 支持按类型（景点/酒店/美食）与城市筛选资源列表；
  - 支持一键评分，直接调用后端评分接口，驱动画像与推荐更新。

- **登录页**（`/login`，`LoginView.vue`）：
  - 用户名 + 密码登录；
  - 登录成功后将用户信息写入 `localStorage`，供推荐等模块读取 `user_id`。

- **注册页**（`/register`，`RegisterView.vue`）：
  - 用户名、密码、邮箱（可选）注册；
  - 处理后端返回的错误信息并提示用户。

- **我的偏好页**（`/profile`，`ProfileView.vue`）：
  - 对话式输入偏好，实时展示 `slots/profile/itinerary`；
  - 登录后可自动写入 `user_profile`，并影响推荐结果；
  - 当 Agent 返回 `itinerary.days` 后，登录用户可直接保存为多日行程；
  - 仍保留手动编辑入口，便于修正偏好。

- **行程规划页**（`/route`，`RouteView.vue`）：
  - 输入城市并加载景点列表，勾选多个景点；
  - 调用 `POST /api/route/plan`，展示推荐游览顺序和估算总路程；
  - 登录用户可直接将结果保存为单日行程，未登录用户仍可正常规划但不能保存。
  - 可调用 `POST /api/route/options` 获取 AMap 路线方案，并查看 `GET /api/route/metrics` 运行指标。

- **我的行程列表页**（`/trips`，`TripsView.vue`）：
  - 展示当前用户已保存的单日与多日行程摘要；
  - 顶部提供“新建空白行程”入口；
  - 显示动态天数、日期范围、总条目数与来源；
  - 支持查看详情、刷新列表和删除行程。

- **空白新建页**（`/trips/new`，`TripCreateView.vue`）：
  - 登录用户可直接创建 `1~10` 天的空白草稿；
  - 表单字段固定为标题、城市、日期、预算、风格和天数；
  - 创建后跳转到详情页继续补内容。

- **行程详情页**（`/trips/:id`，`TripDetailView.vue`）：
  - 支持统一编辑单日与多日行程的标题、日期、出发城市、预算等级、出游风格；
  - 支持空草稿 `0 item` 保存；
  - 支持新增/删除 Day、新增/删除自定义 item；
  - 支持在单个 Day 内调整项目顺序，并编辑每个项目的类型、快照字段、时间、交通方式与备注；
  - 支持通过按钮将条目移动到上一天/下一天，目标 Day 自动接在末尾；
  - 支持在详情页内复用景点/美食/酒店列表接口做资源搜索式选点，可新增资源条目或替换已有条目资源；
  - 支持保存修改和删除行程。

## Trip 编辑器 V3 / V4 补充进展（2026-04-11）

- `/trips/:id` 已完成真实拖拽排序：
  - 同 Day 任意插入；
  - 跨 Day 任意插入；
  - 同时保留按钮式移动作为备选交互。
- `/trips/:id` 已完成 Day 级轻量路线建议与 Day 级真实路线重算：
  - 轻量建议基于本地坐标做最近邻启发式排序；
  - 真实路线重算手动触发，复用 `POST /api/route/options`，固定 `drive + v5`；
  - 前端会对候选顺序串行评分，展示单一最佳方案，并在用户点击“应用到当前 Day”后写回顺序。
- `/trips/:id` 的地图预览仍维持点位 + 编号 + 直线连线，不渲染真实道路 Polyline，也不自动回填时间或交通方式。
- 因此，先前“trip 编辑器尚未支持真实拖拽排序与自动重算路线”的阶段性描述已不再适用；当前尚未覆盖的仍是更高阶能力，例如真实道路几何线渲染、跨天全局联动优化与资源重绑定式路线重算。
