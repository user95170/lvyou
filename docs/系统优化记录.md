## 1. 行程规划算法优化（2025-11-17）

### 优化内容

**背景**：原有行程规划使用简单的最近邻贪心算法，虽然速度快但路径质量不够优化。

**改进方案**：
1. **2-opt 局部优化算法**
   - 在贪心算法基础上进行局部路径改进
   - 通过边交换操作逐步优化路径
   - 设置最大迭代次数（100次）避免过度计算

2. **多目标优化（balanced模式）**
   - 综合考虑距离和景点热度
   - 可配置权重：`distance_weight`（默认0.7）+ `popularity_weight`（默认0.3）
   - 在最短路径和热门景点之间取得平衡

3. **前端策略选择**
   - 贪心算法：最快，适合快速预览
   - 2-opt优化：推荐使用，15-25%路径改进
   - 综合优化：考虑景点热度，适合追求体验

### 技术细节

**核心函数**：
- `_compute_distance_matrix()`：预计算距离矩阵，避免重复计算
- `_optimize_route_2opt()`：2-opt局部优化
- `_compute_route_with_score()`：多目标优化

**2-opt算法原理**：
```python
# 对于路径中的两条边 (i-1, i) 和 (j, j+1)
# 如果交换后 dist(i-1, j) + dist(i, j+1) < dist(i-1, i) + dist(j, j+1)
# 则反转 i 到 j 之间的路径段
```

**API接口变化**：
```json
POST /api/route/plan
{
  "spot_ids": [3, 6, 5, 8],
  "start_spot_id": 3,
  "optimize": "2opt",           // 新增参数
  "distance_weight": 0.7,       // 可选，balanced模式
  "popularity_weight": 0.3      // 可选，balanced模式
}
```

### 预期效果

- **路径长度改进**：2-opt 可改进 15-30%
- **计算时间**：对于 10 个景点，增加 < 100ms
- **用户体验**：提供可选策略，满足不同需求

实现"推荐 → 加入行程 → 一键规划"的完整闭环。
