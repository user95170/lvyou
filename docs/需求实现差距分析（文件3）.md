## 安全补强更新（渐进式鉴权，2026-06-26）

采用**不破坏现有流程**的渐进式方案为登录态增加身份校验：

- ✅ **令牌签发**：登录接口返回签名令牌（`itsdangerous` 基于 `SECRET_KEY`，带时效，无状态不落库）；新增 `services/auth_tokens.py`。
- ✅ **按需鉴权（enforce_user）**：写操作（评分、画像、行为、行程增删改）在**请求携带令牌时**强制校验——无效/过期→401，令牌用户与目标 `user_id` 不一致→403；**未携带令牌则按遗留模式放行**，因此旧客户端与既有测试不受影响。
- ✅ **前端集成**：登录存储令牌，axios 请求拦截器自动附带 `Authorization: Bearer`，401 时自动清除失效令牌；退出登录（`clearCurrentUser`）一并清除令牌。
- ✅ **测试**：新增 `tests/test_auth_token.py`（令牌往返、匹配/不匹配/无效/无令牌、行程越权、匿名行为放行等 8 项），全量 **73** 项后端测试通过；前端构建通过。

> 说明：这是渐进式安全增强（读接口仍开放、无令牌兼容），适合原型平滑过渡；如需更强安全，可后续把读接口与全部写接口改为强制鉴权，并引入刷新令牌/登出失效列表。

## 一、总体结论

- **结论：需求文件《3》中的功能并非全部实现。**
- **整体实现度：约 55%–65%。**
- **强项**：通用推荐框架（Popular/Profile/CF/MF/Hybrid）、多源数据融合（OTA/社交热度）、对话式偏好采集、行程草案生成与行程持久化（CRUD）、监控面板。
- **主要缺口（两条主线）**：
  1. **原始人口特征（性别/年龄/地域/职业）驱动的细粒度个性化** 几乎未打通（无采集入口、主推荐不使用、无差异化提示）。
  2. **资源类型不完整（缺交通/活动/特产）** 且 **缺少统一的"个性化电子地图"主视图**。

## 三、逐模块详细分析

### 模块 1：自然语言理解与用户画像

**✅ 已实现**

- 对话框自然语言输入、多轮对话：`backend/app/routes/agent.py`（`POST /api/agent/chat`，支持 `messages` 多轮、LLM 可选 + 本地规则回退）。
- 意图识别可提取：`目的地(destination)`、`出行时间(days)`、`预算(budget_amount/budget_level)`、`兴趣(interests)`、`出行方式(transport_mode)`、`风格(travel_style)`。

**🟡 部分实现**

- 原始特征作为画像依据：
  - `User` 模型已含 `gender / age / home_region` 字段（`backend/app/models.py`）。
  - 行程草案评分 `itinerary.py` 中 `_score_scenic / _score_food` 使用了 `age`（如 `senior` → 博物馆/公园加权）与 `home_region`（同城微调）；`aggregate_feature_vectors.py` 在聚类特征中纳入 `gender/age_bucket/home_region`。
- 历史行为入画像：评分（`rating`）与行为日志（`user_behavior_log`）已落地，`aggregate_user_profile.py` 基于评分生成偏好类型。

**❌ 未实现 / 缺口**

- **无采集入口**：注册接口 `auth.py` 只收集 `username/password/email/phone`，**不采集性别/年龄/地域**；前端注册页也无对应字段。"职业"字段在模型中不存在。
- **性别加权未真正生效**：`gender` 仅被读入 profile 字典，景点/美食打分未基于性别做"购物/拍照打卡/氛围感"加权（且系统无此类目）。
- **未提取"人数"**：slots 无人数字段。
- **地域口味映射缺失**：无"湖南→辣味/湘菜"这类地域→口味标签规则。
- **交叉特征缺失**：性别×年龄、地域×年龄等组合权重未实现。

## 四、缺口优先级与建议补齐路线

> 以"先打通核心个性化闭环，再扩资源与视图"为原则，分三阶段。

### 阶段一：打通"原始特征 → 画像 → 评分 → 提示"闭环（最高优先）

1. **采集入口**：在注册页/「我的偏好」增加性别、年龄、地域（可选职业）的采集；扩展 `auth.py` 注册与 `user/profile` 接口。
2. **进入主推荐评分**：在 `recommend.py` 的评分中加入基于性别/年龄/地域的权重项（复用 `itinerary.py` 已有的 `_age_group`、`home_region` 逻辑并扩展）。
3. **地域口味规则**：建立"地域→口味/菜系标签"映射（如 湖南→辣/湘菜），影响美食排序。
4. **匹配提示**：在推荐理由中增加"符合您家乡口味""适合偏安静的行程节奏"等基于原始特征的文案。
5. **交叉特征**：加入性别×年龄、地域×年龄的组合规则（如"偏大年龄女性→安静/文化/人少"加权）。

### 阶段二：补齐资源类型与详情

1. 新增 **交通节点 / 活动 / 特产** 数据模型、采集与浏览接口。
2. 完善各类详情字段（电话、评论列表、适配人群标签）。

### 阶段三：统一个性化电子地图与定位路线

1. 新增前端「地图」页面，在地图上展示六类资源、支持点击查看详情与画像匹配标签。
2. 接入**用户实时定位**，实现从当前位置出发的多方案路线（最快/少换乘/步行优先），并在路线优选中纳入画像权重。
