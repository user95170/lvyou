## 1. 系统简介

**内蒙古智慧旅游系统**是一套面向内蒙古自治区旅游场景的智能推荐与出行规划原型系统。
系统整合景点、酒店、美食等多源数据，结合用户画像与行为数据，为用户提供个性化推荐、
相似景点发现、对话式偏好采集、行程规划与行程管理，并配套系统监控面板。

- **目标用户**：游客（普通用户）、系统演示与验收人员、运维管理人员。
- **技术形态**：前后端分离的 Web 应用（后端 Flask REST API + 前端 Vue 3 单页应用）。
- **数据存储**：默认 SQLite（`dev.db`，开箱即用），可切换 MySQL。
- **核心能力**：多源数据融合、混合推荐（Popular / Profile / CF / MF / Hybrid）、
  对话式偏好 Agent、行程规划（贪心 + 2-opt 优化）、行程持久化（CRUD）、运行监控。

## 2. 功能概览

| 模块 | 功能 | 对应页面 / 接口 |
|------|------|------------------|
| 用户系统 | 注册、登录 | `/register`、`/login`、`/api/auth/*` |


```powershell

# 2) 安装依赖
```

```powershell
PS E:\旅游\frontend> npm install
```

## 5. 数据准备

系统的推荐与展示能力依赖基础实体数据、评分数据与多源聚合数据。仓库已内置 `dev.db`，
开箱即可演示。如需重建或补充数据，按下列步骤执行（均在 `backend` 目录、已激活虚拟环境）。

### 5.1 一键准备（SQLite 快速路径）

```powershell
# 1) 生成基础实体（景点/酒店/美食/演示用户），仅在新库或缺数据时需要
(venv) PS E:\旅游\backend> python -m scripts.seed_dev

# 2) 生成评分 + 协同过滤相似度 + 内容聚合（启用 CF/MF/Hybrid 与推荐理由）
(venv) PS E:\旅游\backend> python -m scripts.refresh_dev_data

# 3) （可选）灌入演示用行程与行为日志，便于演示"我的行程"和监控面板非零指标
(venv) PS E:\旅游\backend> python -m scripts.seed_demo_trip_behavior
```

> `seed_demo_trip_behavior` 为幂等脚本，重复执行不会重复插入数据。

### 5.2 多源数据脚本（按需）

| 脚本 | 作用 |
|------|------|
| `python -m app.pipelines.aggregate_content` | 基于内部评分计算内容受欢迎度，写入 `content_standard` |
| `python -m app.pipelines.aggregate_user_profile` | 基于评分行为生成/更新用户画像 |
| `python -m app.pipelines.aggregate_scenic_cf` | 构建 item-based 协同过滤相似度 |
| `python -m app.pipelines.load_ota_data` | 导入 OTA 外部评分/评论（读取 `backend/data/ota_sample.csv`） |
| `python -m app.pipelines.social_media_fetcher` | 从合规导出的社媒 CSV 聚合互动量/情感分 |
| `python -m app.pipelines.fetch_wikipedia_pageviews --lang zh --limit 200 --days 30` | 导入 Wikipedia 浏览量作为社交热度 |
| `python -m app.pipelines.analyze_feature_coverage` | 多源特征覆盖率体检 |
| `python -m app.pipelines.evaluate_recommendation` | 离线推荐评估（Precision@K 等） |

### 5.3 OTA 示例数据格式

`backend/data/ota_sample.csv` 为代表性样本，列结构如下，可按相同格式扩展：

```text
entity_type,entity_name,city,address,source_platform,external_rating,external_review_count,price_range,tags,description
scenic_spot,希拉穆仁草原,呼和浩特,武川县希拉穆仁镇,携程,4.5,1820,免费,草原;骑马;民俗,典型的内蒙古草原观光地
```

- `entity_type`：`scenic_spot` / `hotel` / `food`（导入时 `food` 归一为 `food_place`）。
- 标签 `tags` 以分号 `;` 分隔。

## 6. 启动与访问

### 6.1 启动后端

```powershell
(venv) PS E:\旅游\backend> python .\run.py
```

- 默认监听 `http://127.0.0.1:5000`。
- 健康检查：浏览器访问 `http://127.0.0.1:5000/api/health`，应返回 `{"status": "ok"}`。
- 可通过环境变量调整：`FLASK_RUN_HOST`、`PORT`、`FLASK_DEBUG`（默认关闭调试）。

### 6.2 启动前端

在另一个终端：

```powershell
PS E:\旅游\frontend> npm run dev
```

- 默认访问地址：`http://127.0.0.1:5173/`。
- 前端通过 Vite 代理将 `/api` 请求转发到后端 `http://127.0.0.1:5000`，
  可用环境变量 `VITE_API_PROXY_TARGET` 覆盖后端地址。

> 启动顺序：先启动后端，再启动前端；确保两者同时运行才能正常使用全部功能。

## 7. 功能使用说明

### 7.1 注册与登录

1. 顶部导航点击「注册」，填写用户名（≥3 字符）、密码（≥6 字符）、邮箱（可选），提交。
2. 切换到「登录」，使用刚注册的账号登录。登录信息保存在浏览器本地，用于个性化推荐与行程保存。

### 7.2 目的地浏览与评分

1. 点击「目的地浏览」，选择类型（景点/酒店/美食），可填写城市后搜索。
2. 在景点卡片点击「我喜欢（打 5 分）」进行评分（需登录）。评分会驱动画像与协同过滤。
3. 点击「查看相似景点」可展开基于协同过滤的相似景点列表。

### 7.3 我的偏好（对话式）

1. 点击「我的偏好」，在输入框用自然语言描述需求，例如：
   `想自驾去呼和浩特玩3天，预算3000，亲子`。
2. 系统会返回：助手回复、结构化识别结果（Slots）、以及行程草案预览（Itinerary）。
3. 登录状态下可将识别到的偏好写入画像，并把多日草案保存为行程。
4. 未配置 `LLM_API_KEY` 时，系统回退到本地规则抽取（可用但能力有限）。

### 7.4 首页个性化推荐

1. 点击「推荐」进入首页，查看推荐景点卡片。
2. 通过「策略」下拉框切换算法：
   - **自动**：依据是否有画像自动选择；
   - **popular**：按多源综合得分排序；
   - **profile**：优先展示符合偏好类型的景点；
   - **CF**：基于评分的 item-based 协同过滤；
   - **MF**：矩阵分解预测兴趣分；
   - **Hybrid**：多源基线 + CF + MF 融合（更稳健）。
3. 关注卡片下方「推荐理由」（如"OTA 平台评分优秀""社交媒体热度很高""符合你的偏好类型"）。
4. 冷启动用户（评分/画像过少）会看到引导提示，建议先去浏览页多评分。

### 7.5 行程规划

1. 点击「行程规划」，输入城市并加载景点，勾选 2–5 个景点。
2. 点击「规划路线」，查看推荐游览顺序与估算总里程。
3. 可选优化策略：`greedy`（最近邻贪心）、`2opt`（局部优化）、`balanced`（距离+热度综合）。
4. 配置 `AMAP_WEB_KEY` 后将调用高德路径规划获取更精确距离，否则使用经纬度近似。

### 7.6 我的行程（保存与管理）

1. 登录后可从「行程规划」保存单日路线，或从「我的偏好」保存多日草案。
2. 「我的行程」列表展示标题、日期、天数、条目数与来源。
3. 详情页（`/trips/:id`）支持：新增/删除 Day、新增自定义或资源型条目、
   跨天移动条目、从资源库搜索选点、按 Day 进行真实路线重算与地图预览。
4. 「新建空白行程」可创建只含天数骨架的草稿，逐步填充。
5. 支持删除行程（含级联清理子表）。

### 7.7 监控面板

1. 点击「监控」进入 `/monitor`，查看：
   - **系统状态**：健康检查结果（`ok`）；
   - **资源数据**：景点/酒店/美食/聚合内容数量；
   - **用户与交互**：用户、画像、评分、行为日志、行程数量；
   - **推荐运行**：推荐请求数、策略分布、回退统计；
   - **路线服务**：调用成功率、缓存命中、回退等指标。
2. 点击「刷新」可重新拉取最新指标。

| 变量 | 默认值 | 说明 |
|------|--------|------|

## 10. 常见问题与排查（FAQ）

1. **前端白屏 / 接口 404**
   - 确认后端已启动且 `http://127.0.0.1:5000/api/health` 返回 `ok`；
   - 确认前端通过 `npm run dev` 启动，访问 `http://127.0.0.1:5173/`。

2. **推荐列表为空或无推荐理由**
   - 运行 `python -m scripts.refresh_dev_data` 生成评分/CF/聚合；
   - 检查 `content_standard` 是否存在 `internal_rating`/`ota_stats`/`social_media`/`item_cf_similarity` 记录。

3. **相似景点为空**
   - 确认已运行 `aggregate_scenic_cf` 且 `rating` 表有景点评分。

4. **行程规划结果为空**
   - 确认所选城市在 `scenic_spot` 表中有数据，且至少勾选 2 个景点。

5. **行程无法保存**
   - 确认已登录；`title/origin_city/days` 已填且 `days` 在 1–10；`trip_days` 为合法数组。

6. **对话式偏好识别不准**
   - 未配置 `LLM_API_KEY` 时仅用本地规则抽取，建议配置 LLM 以提升识别质量。

7. **监控面板行程/行为指标为 0**
   - 运行 `python -m scripts.seed_demo_trip_behavior` 灌入演示数据。

8. **数据库相关报错**
   - 默认 SQLite 无需配置；若用 MySQL，确认 `DATABASE_URL` 正确且数据库可连接。

### 11.3 术语表

| 术语 | 含义 |
|------|------|
| CF | Collaborative Filtering，协同过滤 |
| MF | Matrix Factorization，矩阵分解 |
| Hybrid | 多源基线 + CF + MF 的混合推荐策略 |
| OTA | Online Travel Agency，在线旅游平台（携程/美团等） |
| 画像（Profile） | 用户偏好的结构化表示 |
| 冷启动 | 用户评分/画像不足，难以个性化推荐的初始状态 |
| Slots | 对话中抽取出的结构化字段（目的地/天数/预算等） |
